HTML and CSS Cheat Sheet

CSS Selectors
*Listed from highest to lowest specificity.

Selector Name CSS Syntax Description Example
ID Selector #IDname { }

A hash (#) charater followed by the name of the ID.

 Selects elements with a specific ID attribute.

HTML:

<section id='jounal'>

Text of the jounal entry ...</p>

CSS:

#jounal { font-family: cursive; }

Result:

Text of the jounal entry ...
Class Selector .classname { }

A period (.) charater followed by the name of the class.

 Selects elements with a specific class attribute.

HTML:

<p class='brand'>

Sole Shoe Company</p>

CSS:

.brand { text-transform: uppercase; }

Result:

SOLE SHOE COMPANY
Type Selector
(aka Element Type Selector)		 element { } Selects elements based on the html element type (e.g. <h1>, <p>, <body>).

HTML:

<h1>This is the text for H1.</h1>

<p>This is the text for p.</p>

CSS:

h1 { color: blue; }

Result:

This is the text for H1.

This is the text for p.
Universal Selector * { } Selects all elements of any type.

HTML:

<h1>This is the text for H1.</h1>

<p>This is the text for p.</p>

CSS:

* { color: blue; }

Result:

This is the text for H1.

This is the text for p.

CSS Specificity Rules

# Rule
Rule 1: The CSS selector with the higher specificity applies.
Rule 2: If CSS selectors have equal specificity, then the last rule in the document applies.
Rule 3: Inline CSS has the highest specificity.
Rule 4: the !important property overrides all other declarations on an element including inlne declarations.
